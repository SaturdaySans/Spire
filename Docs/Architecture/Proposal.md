# Proposal

## Problem Statement

> Default browser landing pages are boring and chopped

Many users rely on browser home pages such as Google’s default landing page. However, these pages are often minimal, static, and lowkey just ugly. As a result, users frequently open multiple tabs or external applications to access simple information such as schedules, reminders, or quick tools.

This leads to reduced productivity and a fragmented browsing experience.

## Functionality

- Similar to distro ricing
- Replace the default browser home page with a personalised dashboard
- Add and manage functional widgets such as reminders, timetable, and quick notes
- Dynamically create, update, and delete widgets (Users can write their own widgets in JS)
- Maintain a visually minimalistic and aesthetically pleasing interface (vvv important)
- Optimise performance for low memory usage and fast loading

## Key Features

```
Custom Widget System
- Users can add, remove, and configure widgets
- Support for user-created widgets written in JavaScript

Personal Dashboard
- Centralised homepage for daily utilities
- Quick access to commonly used tools

Lightweight Design
- Minimal resource usage
- Fast load times and efficient rendering

Responsive UI
- Adaptable layout for different screen sizes
- Clean and consistent visual hierarchy

Pretty backgrounds
- Self explanatory
```

## User Interactions

Users will be able to:
- Create, edit, delete widgets
- Arrange and drag widgets on a dashboard grid
- Save personalised layouts
- Interact with widgets (e.g. reminders, timetable viewing, notes)
